# 🖥️ 프로젝트 주제

본 프로젝트는 가상 환경에서 Active Directory 기반 기업 내부망을 직접 구축한 뒤,
계정 탈취 - 권한 상승 - 내부 이동/관리자 권한 획득 흐름의 공격을 재현하고
그 과정에서 발생하는 Windows 이벤트 로그를 수집/분석하여 탐지와 방어 시나리오를 적용하는
실습형 보안 랩을 구현하는 것을 목표로 한다.

👉 인프라 구축 → 공격 재현 → 로그 수집 및 탐지 → 방어 정책 적용

## **SOAR-lite (Security Orchestration, Automation, and Response)**

👉 보안 이벤트가 발생했을 때, 사람이 일일히 처리하지 않도록 일부 절차를 자동화

- 경고 발생 시 관련 로그 자동 수집
- 위험도 높은 이벤트면 슬랙 등으로 알림 전송
- 이벤트 발생 계정 정보 자동 조회
- 대응 체크리스트 자동 출력

-의 간소화 버전으로 구현하는 것을 목표로 함

## 선정 이유 및 기대 효과

1. 실제 기업 환경에서 사용되는 AD의 구조를 이해할 수 있다.
2. 공격과 방어를 함께 다룰 수 있다.
3. Windows 보안 로그 분석 경험을 쌓을 수 있다.

# 🪧 프로젝트 목표

## 1. 인프라 구축

- VM 기반의 독립적 AD 실습 환경 구축
- 사용자, 관리자, 서비스 계정, 그룹, OU 구성
- VMWare : Windows 서버(AD) , Windows 클라이언트(AD), Linux 공격 머신(Kali)
- Docker : 로그 수집, 분석, 백엔드, 대시보드 등

## 2. 공격 시나리오 재현

- 대표적인 AD 공격 시나리오 2~3개 선정
- (ex) 계정 탈취, Kerberos 기반 공격, 권한 상승, 내부 이동 흐름 구성

## 3. 로그 수집 및 탐지

- 공격 과정에서 남는 주요 이벤트 로그 수집
- (ex) 이벤트 ID, 시간, 계정, 대상 시스템을 연결한 타임라인 작성

## 4. 방어 시나리오 적용

- (ex) 계정 정책, 권한 분리, 관리자 사용 제한, GPO, 감사 정책 강화
- 방어 적용 전후 동일 공격 재실행

## 5. 대시보드 및 시각화

- 위험도 점수화 및 분류 (ATT&CK 매핑)
- 슬랙 등 알림 연동
- 설명형 AI

## 6. 고도화 (후반부 여유 있을 시)

- 웹 공격 확장

# 🗞️ 시나리오

## ⚔️ 공격 시나리오

### 1. 초기 접근 / 계정 정보 탈취

- LLMNR/NBT-NS 포이즈닝 또는 유사한 계정 정보 수집 흐름
- 일반 사용자 계정 또는 인증 해쉬 확보

### 2. Kerberoasting 기반 권한 상승

- 서비스 계정 티켓을 요청해 오프라인 비밀번호 크래킹 시도
- 취약한 서비스 계정 비밀번호 탈취 및 상위 권한 확보

### 3. 비정상 로그인 / 권한 변경 / 관리자 권한 확득

- 관리자 그룹 변경
- 특권 계정 로그인
- 원격 접속 또는 도메인 관리자 권한 사용 흔적 분석

## 🔍 탐지/분석 시나리오

- 로그인 실패 횟수 급증
- 비정상 시간대 로그인
- 짧은 시간 안에 여러 PC에서 동일 계정 로그인 시도
- 일반 사용자의 관리자 그룹 추가
- Kerberos 티켓 요청 급증
- 특권 계정 사용시 알림

## 🛡️ 방어 시나리오

- LLMNR, NetBIOS 비활성화
- 서비스 계정 비밀번호 복잡도 강화 및 주기적 변경
- 관리자 계정 사용 제한
- 관리자 그룹 변경 감시
- GPO를 통한 감사 정책 강화
- 특권 계정 이벤트 알림
- 필요시 Sysmon, Winlogbeat 기반 수집 강화
